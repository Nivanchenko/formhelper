# formhelper

Библиотека для изменения форм 1С Предприятия из программного кода 1С Прдприятия

Позволяет удобно вносить изменения в типовые конфигурации без необходимости снимать с поддержки формы объектов.

## Описание

Легко и непринужденно изменяем реквизиты и элементы форм в функциональном стиле:

```bsl
РаботаСФормамиСервер.НовыйПостроительФорм(ЭтаФорма)

    // Шаг 1. Добавлен новый реквизит на форму
    .РеквизитСтрока("РеквизитСтрока", 150)

    // Шаг 2. Использован реквизит из предыдущего шага для добавления ного элемента формы
    .ПолеВвода("РеквизитСтрока")
        .Заголовок("Очень полезный реквизит")
        .ПутьКДанным("РеквизитСтрока")

    // Шаг 3. Финализируем изменения
    .Применить()
;
```

В итоге у нас получается код, который не зависит от изменений в последующих обновлениях конфигурации и который легко читать.

## Усложняем задачу (условия)

Ничего не мешает строить интерфейс с учетом какой-либо бизнес-логики.

```bsl
// Сохраняем обработку, которая хранит в себе изменения в форме и осуществляет
// непосредственно изменения в отдельной переменной. Для того чтобы использовать
// эту обработку в дальнейших манипуляциях
ПостроительФорм = РаботаСФормамиСервер.НовыйПостроительФорм(ЭтаФорма);

// Условие, определяющее как будет выглядеть форма
Если ПолучитьФункциональнуюОпцию("ИспользоватьНашуБизнесЛогику") Тогда

    ПостроительФорм.
        // Шаг 1а. Добавлен новый строковый реквизит на форму
        .РеквизитСтрока("РеквизитБизнесЛогики")
    ;

Иначе
    
    ПостроительФорм.
        // Шаг 1б. Добавлен новый числовой реквизит на форму
        .РеквизитЧисло("РеквизитБизнесЛогики")
    ;

КонецЕсли;

// Дальше, по сравнению с предыдущим примером, ничего не изменилось
ПостроительФорм.
    // Шаг 2. Использован реквизит из предыдущего шага для добавления ного элемента формы
    .ПолеВвода("РеквизитБизнесЛогики")
        .Заголовок("Очень полезный реквизит")
        .ПутьКДанным("РеквизитБизнесЛогики")

    // Шаг 3. Финализируем изменения
    .Применить()
;
```

Следующим этапом предлагаю ознакомиться с тем, как можно начать использовать функционал этой библиотеки, и забыть о тех болях, которые приносят нам обновления типовых конфигураций 1С: [начало работы](/docs/getting-started/).
